## Codebase Patterns
- Use `bun:sqlite` (`import { Database } from 'bun:sqlite'`) for all SQLite access — no external packages
- All schema migrations use `CREATE TABLE IF NOT EXISTS` for idempotency
- DB module lives at `src/db.ts` and exports the `db` instance; import with `import "./db"` in `index.ts` to ensure it runs on startup
- API route modules export a `const routes = { ... } as const` object that gets spread into `Bun.serve({ routes: { ...routeModule } })`
- With `noUncheckedIndexedAccess: true`, use `req.params["id"] ?? ""` instead of `req.params.id` to avoid TS errors on indexed access
- Use `db.prepare<Row, Params>("... RETURNING ...").get(...)` for INSERT/UPDATE that need the inserted/updated row back
